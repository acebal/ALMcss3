ALMCSS - Parser Test

Currently the parser is very simplistic (not so the lexical analyser): it only recognised rule sets and, from those rules, only display and position properties are syntactically checked and returned in a sort of simplified CSS Object Model.

However it should be able not only to read any other CSS rule, but also recover both for true CSS errors and for those other parts of the grammar that have not been implemented yet (for example, “import” rules).

The HTML files in this folder try to test only that thing: the parser, to check that errors do not force infinite loops and things like that.
